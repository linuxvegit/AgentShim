# Plan 02 — DeepSeek Native Adapter

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Source design:** [`docs/superpowers/specs/2026-04-30-phase-2-provider-breadth-design.md`](../specs/2026-04-30-phase-2-provider-breadth-design.md) (decisions D3, D4, D7).

**Goal:** Land the DeepSeek native adapter, composing the `oai_chat_wire/` lib introduced in Plan 01 and adding the three DeepSeek-specific behaviors: `reasoning_content` parsing into interleaved reasoning blocks, prompt-cache usage mapping, and the `interleaved_reasoning` state machine that Plan 03's Gemini adapter will reuse.

**Architecture:**
- New sibling provider module `crates/providers/src/deepseek/` that composes `oai_chat_wire`. The bulk of request encoding and SSE parsing is delegated; DeepSeek's specifics live in three places: a delta-extension hook on the SSE parser to extract `reasoning_content`, a usage mapper post-processing the unary/final usage, and the state machine that emits `ContentBlockStart`/`ContentBlockStop` boundaries when reasoning↔content transitions happen.
- The `interleaved_reasoning.rs` state machine ships in `oai_chat_wire/` (not `deepseek/`), so Gemini can import it from one place. Plan 03 consumes; Plan 02 produces.
- Configuration: a new `type: deepseek` upstream variant. Inherits OAI-compat shape but defaults `base_url: https://api.deepseek.com/v1` and sets `reasoning_capable: true`.

**Tech stack:** No new dependencies. Reuses `reqwest`, `serde_json`, `tokio`, `bytes`. Tests: `mockito` + inline canonical-event assertions for cross-protocol.

**Core changes:** NONE (frozen-core, ADR-0002). DeepSeek `provider_raw` payloads land in the existing `Usage.provider_raw` field. Reasoning interleaving uses existing `ContentBlock::Reasoning`.

## Tasks

### Task 1: `interleaved_reasoning.rs` state machine

- [ ] Create `crates/providers/src/oai_chat_wire/interleaved_reasoning.rs` with a `ReasoningInterleaver` struct that consumes a stream of (text or reasoning) deltas tagged with their kind, and produces correctly-ordered `StreamEvent`s.

```rust
pub enum DeltaKind { Text, Reasoning }

pub struct ReasoningInterleaver {
    state: State,
    next_index: u32,
}

enum State { Idle, InText { index: u32 }, InReasoning { index: u32 } }

impl ReasoningInterleaver {
    pub fn new() -> Self { ... }

    /// Push a delta. Emits the right combination of ContentBlockStart/Stop and
    /// TextDelta/ReasoningDelta events into `out`.
    pub fn push(&mut self, kind: DeltaKind, text: &str, out: &mut Vec<StreamEvent>);

    /// Close the current block (no-op if Idle). Called on stream end.
    pub fn flush(&mut self, out: &mut Vec<StreamEvent>);
}
```

- [ ] Unit tests: empty stream, all-text, all-reasoning, alternating, three text → reasoning → text transitions, flush closes the open block. Co-located in the same file via `#[cfg(test)] mod tests`.

### Task 4: SSE parser extension for `reasoning_content`

- [ ] In `crates/providers/src/deepseek/response.rs`, define a parser wrapping `oai_chat_wire::chat_sse_parser` that:
  1. Reads each chunk's `delta` field.
  2. If `delta.reasoning_content` is `Some(text)`, push (DeltaKind::Reasoning, text) into the `ReasoningInterleaver`.
  3. If `delta.content` is `Some(text)`, push (DeltaKind::Text, text) into the interleaver.
  4. Drain the interleaver's emitted events into the canonical stream.
- [ ] On `finish_reason` arrival, flush the interleaver, emit `MessageStop` + `ResponseStop`.
- [ ] Implementation note: `oai_chat_wire::chat_sse_parser` may need a small extension hook (a closure or trait method) for "examine each delta for extra fields." If extending `chat_sse_parser` complicates the OAI-compat caller, instead **clone-and-modify** the parser into `deepseek/response.rs` — duplicate parsing is cheaper than a wrong abstraction. **Decision deferred to implementation time** based on actual code shape.
- [ ] Test: feed a fixture SSE with mixed `reasoning_content` and `content` deltas, assert canonical stream has interleaved `ContentBlock::Reasoning` and `ContentBlock::Text` blocks in the right order.

### Task 5: Cache usage mapping

- [ ] In `crates/providers/src/deepseek/usage.rs`, write:

```rust
pub fn map_usage(raw: &serde_json::Value) -> Usage {
    let prompt_tokens = raw.get("prompt_tokens").and_then(Value::as_u64).map(|n| n as u32);
    let completion_tokens = raw.get("completion_tokens").and_then(Value::as_u64).map(|n| n as u32);
    let cache_hit = raw.get("prompt_cache_hit_tokens").and_then(Value::as_u64).map(|n| n as u32);
    let cache_miss = raw.get("prompt_cache_miss_tokens").and_then(Value::as_u64).map(|n| n as u32);

    Usage {
        // input_tokens reflects the *new* (non-cached) prompt input
        input_tokens: cache_miss.or(prompt_tokens),
        output_tokens: completion_tokens,
        cache_creation_input_tokens: None,        // DeepSeek has no concept of cache creation
        cache_read_input_tokens: cache_hit,
        reasoning_tokens: None,                    // DeepSeek doesn't separately report; future work
        estimated: false,
        provider_raw: Some(raw.clone()),
    }
}
```

- [ ] Wire `map_usage` into both the unary parser and the SSE final-chunk handler.
- [ ] Inbound `cache_control` from the canonical request: log at `debug!` and drop. Implement in `request::build` post-processing — strip extension keys matching `anthropic.cache_control` before serializing.
- [ ] Unit tests: cache_hit set + cache_miss set, only prompt_tokens set, missing fields all map to None.
